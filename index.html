Evolving Information

Our system adapts at runtime as new information, behaviors, and models arise, and the program changes shape to fit the data instead of forcing the data to fit the program. Any type of record can be modified in both content and structure to meet changing needs at any time.

Ours is a new kind of data architecture–using an adaptable information management system, we’re opening up new possibilities in areas like business operations, resource management, and collaborative investigation. Oakleon is looking for talented developers to push this technology forward.

An open problem

Today there are thousands of business applications for managing information, each of them ineffective for varying reasons. Data management applications often have overlaps in functionality, which can lead to inefficiencies in training, redundancies in data entries, and circular dependencies between systems. A generalized approach to gathering, storing, and reporting structured information is needed, and we are creating it.

Solutions

Project Oakfield

Designed to solve the problem of storing heterogeneous structured records, Oakfield delivers near real-time user updates, eliminating issues like lost entries, inaccurate data, or redundancies. Because Oakfield is an adaptable database structure, schema descriptions are not required before data curation or entry. Users have the ability to iterate on the structure of the data over time, shifting the data structure to meet the needs of the present.

We use open source technologies as our foundation, keeping ties to any proprietary vendors minimal. Individual components will be updated as new systems develop, keeping our solutions agile and constantly expanding in applicability.

Oakfield’s core components include a distributed cache, graph database, full-text search, Isotope (our custom protocol), and worker cluster.

Oakfield API example

// create a record
r1 = {
    id:         1,
    dob:        "3/14/1592",
    type:       "user",
    name:       "Leonhard Euler",
    tags:       ["tech", "manager"]
    number:     "196-9665",
    namespace:  "acme001"
}
// commit the record
Oakfield.commit([r1])

// Link heterogeneous records together

// some equipment data
e1 = {..., type:"vehicle" }

// some sensor data
s1 = {..., geo:[32.80, -117.17]}

// Link the records with tags
Oakfield.link(r1, e1, ["uses"]);
Oakfield.link(e1, s1, ["tracks"]);
Oakfield.commit([r1,e1,s1]);

// Modify
r1 = Oakfiled.query.id(1);
f1 = {status:"active"};
Oakfield.append(f1, r1);

// add a field to many records
records = Oakfiled.query.tag("user");
f1 = {status:"active"};
Oakfield.append(f1, records);

// Query
Oakfiled.query.tag("user");
Oakfiled.query.geo(33.02, -117.41, 5);
Oakfiled.query.graph("(:user)->(:friends)");
Oakfiled.query.fulltext("euler");

Project Canopy

This is our adaptable user interface, capable of automating interface design processes. Each element updates dynamically in response to the data it is asked to present. By using common use patterns for different types of applications, Canopy suits the needs of the task. There’s no building input forms or boilerplate endpoints. This UI builds itself.

Canopy API example

// create list UI structure with a query
v1 = {
    viewtype:   "list",
    querytype:  "graph",
    p1:         "customer",
    p2:         "service",
    p3:         timespan(30,"days"),
    p4:         now - p3;
    q: (p1) -> (p2 .date < p4)
}
// push the view onto the stack
Canopy.viewstacks[0].push(v1);

// the UI is updated automatically
// and the view is kept up to date

// add a field
r1 = Oakfiled.query.id("14794b5df8f");
f1 = {status:"active"};
Oakfield.append(f1, r1);

// add a field to many records
records = Oakfiled.query.tag("user");
f1 = {status:"active"};
Oakfield.append(f1, records);

// every client viewing these records 
// will be updated

// timeline
v1 = {
    viewtype:   "timeline",
    querytype:  "tag",
    q:          "events"
}

// push the view onto the stack
Canopy.viewstacks[0].push(v1);

// the UI is updated automatically
// and the view is kept up to date

// map
v1 = {
    viewtype:   "map",
    querytype:  "geobox",
    p1:         [33.0345,-117.8234],
    p2:         [33.1575,-117.6004],
    q:          p1,p2
}
// push the view onto the stack
Canopy.viewstacks[0].push(v1);

// the UI is updated automatically
