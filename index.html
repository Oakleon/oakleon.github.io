OAKLEON

Distributed version control for structured information

Evolving Information

Oakleon is creating a new kind of data architecture. Our system adapts on the fly as information, behaviors, and disruptive models arise. Put another way, the program changes shape to fit the data instead forcing the data to fit the program. Any kind of record can be forked, merged, committed, and rolled back, like distributed version control for structured data. This opens up all kinds of possibilities in areas like business operations, resource management, and collaborative investigation. Oakleon is looking for talented developers to push this technology forward.

An open problem

Today there are thousands of business applications for managing information. Everything from customer lists and legal forms to experimental research, each with a slightly different way of doing the most basic tasks. Yet each of these programs do many of the same things, present forms for users to fill in, provide query operations, and display results in a report. Steadily these programs grow in complexity and scope until there is significant overlap in functionality. This leads to problems with training users on many programs, double entry, and even a circular dependency of systems. A generalized approach to gathering, storing, and reporting structured information is needed.

Our Solution

Oakfield is a system of systems designed to solve the problem of storing heterogeneous structured records with near real-time user updates. Oakfield should overcome the need to describe a schema before data curation and entry thereby allowing the users to iterate on the structure over time. This system uses modern open source data systems as a foundation so individual components can be updated and modified as new technologies develop without lock-in to a proprietary vendor. Oakfields core components include a distributed cache, graph database, full-text search, Isotope protocol, and worker cluster.

Project Oakfield API example


// create a record
r1 = {
    id:         1,
    dob:        "3/14/1592",
    type:       "user",
    name:       "john doe",
    tags:       ["tech", "manager"]
    number:     "196-9665",
    namespace:  "acme001"
}
// save it
Oakfield.commit([r1])



// Link heterogeneous records together

// some equipment data
e1 = {..., type:"vehicle" }

// some sensor data
s1 = {..., geo:[32.80, -117.17]}

// Link the records and add some tags
Oakfield.link(r1, e1, ["assisgned"]);

Oakfield.link(e1, s1, ["tracks"]);
